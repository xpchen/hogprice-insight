## 缺口说明

- **缺尾**：库内最晚日期早于该年「腊月二十八」阳历日，即年末几天无数据（多为 2～3 天）。
- **缺首**：仅 2024 年库内最早为 2024-02-18，比正月初八 2024-02-17 少 1 天。

## 图表是否正确

- 接口已按「农历年 = 正月初八～腊月二十八 的阳历区间」查询，图表展示的即是该区间内**库内实际有的**数据。
- 因此图表逻辑正确；若某年曲线未到腊月二十八或未从正月初八起，是因为**数据源（涌益 Excel）在该区间内本身缺这些日期**，而非查询或前端画图错误。
- 若需各年“完整”到腊月二十八，需在数据源/导入中补全对应日期的记录。

## 如何复现核对

在项目 backend 目录下执行：

```bash
python scripts/check_slaughter_lunar_year_ranges.py
```

（Windows 下可先设置 `PYTHONIOENCODING=utf-8` 以便正确显示中文。）
